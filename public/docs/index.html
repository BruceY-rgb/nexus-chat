API Documentation

Welcome to the Slack-like Chat Application API

This is a complete RESTful API that supports real-time messaging, channel management, user authentication, and more.

Authentication

This API uses JWT Token for authentication. After successful login, the system sets auth_token in the Cookie.

# Set authentication Header
curl -H "Cookie: auth_token=your_token_here" \
     http://localhost:3000/api/auth/me

Note: All authenticated APIs require a valid auth_token Cookie in the request.

Response Format

Success Response

{
  "success": true,
  "data": { ... },
  "message": "Operation successful"
}

Error Response

{
  "success": false,
  "error": "Error message",
  "code": "ERROR_CODE"
}

HTTP Status Codes

Status Code Description Example
200 Success Request completed successfully
400 Bad Request Missing required parameters
401 Unauthorized No valid authentication token provided
403 Forbidden Insufficient user permissions
404 Not Found Requested resource not found
500 Internal Server Error Error occurred while processing the request

Authentication API

POST /api/auth/login

User login endpoint, supporting both password and verification code login methods.

Request Parameters

Parameter Type Required Description
email string Yes User email
password string No* User password (required for password login)
code string No* Verification code (required for verification code login)

*Either password login or verification code login is required

Request Example

# Password login
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "password123"
  }'

# Verification code login
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "code": "123456"
  }'

Response Example

{
  "success": true,
  "data": {
    "user": {
      "id": "123",
      "email": "user@example.com",
      "displayName": "John Doe",
      "realName": "John Doe",
      "avatarUrl": "https://example.com/avatar.jpg",
      "status": "active",
      "role": "member",
      "isOnline": true,
      "notificationSettings": {
        "emailNotifications": true,
        "pushNotifications": true
      }
    }
  },
  "message": "Login successful"
}

Cookie Settings: After successful login, the system sets two Cookies:

  • auth_token (httpOnly) - Used for API authentication
  • ws_token (non httpOnly) - Used for WebSocket authentication

Messages API

POST /api/messages

Send a new message to a channel or direct message.

Request Parameters

Parameter Type Required Description
content string No* Message content
channelId string No* Channel ID (choose one with dmConversationId)
dmConversationId string No* Direct message conversation ID (choose one with channelId)
attachments array No Array of attachments

*The message must have either text content or attachments

Request Example

curl -X POST http://localhost:3000/api/messages \
  -H "Content-Type: application/json" \
  -H "Cookie: auth_token=your_token_here" \
  -d '{
    "content": "Hello everyone!",
    "channelId": "channel123"
  }'

  "success": true,

    "updatedAt": "2024-01-01T12:00:00.000Z",

  }
}

Quick Start

1. Start the MCP Server

cd /nexus-chat/mcp-server
npm run build
node dist/index.js

2. Register a New User

{
  "name": "register",
  "arguments": {
    "email": "user@example.com",
    "password": "securepassword123",
    "displayName": "John Doe"
  }
}

3. Login

{
  "name": "login",
  "arguments": {
    "email": "user@example.com",
    "password": "securepassword123"
  }
}

4. Use Other Tools

After login, use other tools to manage channels, messages, and users:

{
  "name": "list_channels",
  "arguments": {
    "type": "joined"
  }
}
{
  "name": "send_message",
  "arguments": {
    "content": "Hello!",
    "channelId": "channel123"
  }
}

Error Handling

All tool responses follow a consistent format:

{
  "success": false,
  "error": "Error message description"
}

Common error codes:

  • UNAUTHORIZED: User not logged in or token expired
  • NOT_FOUND: Requested resource not found
  • FORBIDDEN: User lacks permission
  • VALIDATION_ERROR: Invalid input parameters
  • INTERNAL_ERROR: Server error

Resources

The MCP server also provides resources for querying data:

URI Description
channels://list List of all channels
channels://joined List of joined channels
messages://recent Recent messages
users://list List of all users

Example Code

JavaScript (Node.js)

// Login
const login = async (email, password) => {
  const response = await fetch('http://localhost:3000/api/auth/login', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ email, password }),
    credentials: 'include' // Include Cookie
  });
  return response.json();
};

// Send Message
const sendMessage = async (content, channelId) => {
  const response = await fetch('http://localhost:3000/api/messages', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Cookie': 'auth_token=your_token_here'
    },
    body: JSON.stringify({ content, channelId }),
    credentials: 'include'
  });
  return response.json();
};

Python

import requests

# Login
def login(email, password):
    response = requests.post(
        'http://localhost:3000/api/auth/login',
        json={'email': email, 'password': password},
        cookies={'auth_token': 'your_token_here'}
    )
    return response.json()

# Send Message
def send_message(content, channel_id):
    response = requests.post(
        'http://localhost:3000/api/messages',
        json={'content': content, 'channelId': channel_id},
        cookies={'auth_token': 'your_token_here'}
    )
    return response.json()
